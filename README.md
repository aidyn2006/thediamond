# TheDiamond

Веб-платформа, соединяющая бренды Казахстана с UGC-креаторами и микроинфлюенсерами.
Бренды публикуют кампании, креаторы откликаются, выполняют задание и сдают ссылку на
опубликованный контент. Оплата в v1.0 — вне платформы.

## Архитектура

Монорепо из двух приложений:

```
/backend    Spring Boot 4 (Java 21+) REST API — JPA/Hibernate, Flyway, Spring Security + JWT
/frontend   Next.js 16 (App Router, TypeScript) — Tailwind v4, Auth.js, zod
docker-compose.yml   PostgreSQL 16
```

Фронт обращается к бэкенду по HTTP. Авторизация: Auth.js (credentials) на фронте
делегирует проверку логина в Java-бэкенд, который владеет паролями (BCrypt) и выдаёт
JWT; токен хранится в httpOnly-сессии Auth.js и прокидывается `Bearer`-ом в API.

> Примечание. ТЗ фиксировало full-stack на Next.js + Prisma + Auth.js; по требованию
> заказчика бэкенд реализован на Java (Spring Boot). Как следствие: Prisma → JPA/Hibernate +
> Flyway, серверная валидация zod → Jakarta Bean Validation (zod остался на клиенте),
> отправка почты и хранение файлов — на стороне Java. Модель данных, статус-машины,
> правила доступа, экраны и дизайн-токены — строго по ТЗ.

## Требования

- JDK 21+ (собрано и проверено на OpenJDK 25)
- Node.js 20+ (проверено на 24) и npm
- Docker + Docker Compose

## Локальный запуск

### 1. База данных

```bash
docker compose up -d
```

Поднимает PostgreSQL на **localhost:5433** (порт 5432 часто занят; при желании
поменяйте маппинг в `docker-compose.yml`). БД/логин/пароль: `thediamond` / `thediamond` / `thediamond`.

### 2. Бэкенд

```bash
cd backend
# Windows (пример пути к JDK):
JAVA_HOME=/c/Users/<you>/.jdks/ms-25.0.3 ./mvnw spring-boot:run
# macOS/Linux:
./mvnw spring-boot:run
```

Стартует на **http://localhost:8080**. При первом запуске Flyway создаёт схему и
заливаются сиды (1 админ, 5 креаторов, 2 бренда, 3 кампании).

### 3. Фронтенд

```bash
cd frontend
cp .env.example .env.local   # и укажите свой AUTH_SECRET
npm install
npm run dev
```

Открывается на **http://localhost:3000**.

## Переменные окружения

### Backend (`backend`, через env; есть дефолты для локали)

| Переменная | Назначение | Дефолт |
|---|---|---|
| `DATABASE_URL` | JDBC-строка Postgres | `jdbc:postgresql://localhost:5433/thediamond` |
| `DATABASE_USER` / `DATABASE_PASSWORD` | доступ к БД | `thediamond` / `thediamond` |
| `JWT_SECRET` | секрет подписи JWT (≥32 символов) | dev-дефолт (заменить в проде!) |
| `JWT_TTL_MINUTES` | время жизни токена | `720` |
| `CORS_ALLOWED_ORIGINS` | разрешённые origin фронта | `http://localhost:3000` |
| `UPLOAD_DIR` | папка аватаров | `./uploads` |
| `UPLOAD_PUBLIC_BASE_URL` | публичный URL файлов | `http://localhost:8080/uploads` |
| `RESEND_API_KEY` | ключ Resend для писем (без него — лог в консоль) | пусто |
| `MAIL_FROM` | адрес отправителя | `TheDiamond <onboarding@resend.dev>` |

### Frontend (`frontend/.env.local`)

| Переменная | Назначение |
|---|---|
| `AUTH_SECRET` | шифрование сессии Auth.js (`openssl rand -base64 32`) |
| `AUTH_TRUST_HOST` | `true` для локали/прод-хоста |
| `BACKEND_INTERNAL_URL` | базовый URL бэкенда (серверные вызовы) |

## Деплой

- **Фронтенд → Vercel.** Импортируйте `frontend/`. Env: `AUTH_SECRET`, `AUTH_TRUST_HOST=true`,
  `BACKEND_INTERNAL_URL` (публичный URL бэкенда). Разрешите домен фронта в `NEXT_PUBLIC`… не нужен —
  все вызовы серверные.
- **БД → Neon / Supabase.** Создайте Postgres, возьмите строку подключения.
- **Бэкенд → Railway / Render / Fly.io** (Vercel не хостит Spring Boot). Env: `DATABASE_URL`
  (Neon), `JWT_SECRET`, `CORS_ALLOWED_ORIGINS` (домен Vercel), при необходимости `RESEND_API_KEY`.
  Файлы-аватары в облаке лучше вынести на S3-совместимое хранилище (реализация за интерфейсом
  `StorageService`).

## Известные ограничения v1.0

- Оплата — вне платформы (нет кошелька/эквайринга) — по ТЗ.
- Аватары хранятся на локальном диске бэкенда; для облака нужна S3-реализация `StorageService`.
- Автозакрытие кампаний по дедлайну — «ленивое» (при чтении ленты/детали), без фонового
  шедулера.
- Мобильная навигация — единый нижний таб-бар для обеих ролей (в ТЗ для бренда предполагался
  бургер; бренд-флоу десктоп-приоритетный).
- Гранёная «D» в логотипе — по присланному брендбуку (SVG), в акцентном цвете для читаемости
  на тёмной теме.
- Нет мультиязычности (строки RU, i18n-ready), OAuth, соц-API, чата, рейтингов — по разделу 1.8 ТЗ.
